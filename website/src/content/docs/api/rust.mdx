## Derive Macros

### `#[derive(Event)]`

Implements the `Event` trait on a struct. Must be combined with `Serialize` and `Deserialize`.

```rust
#[derive(Debug, Clone, Serialize, Deserialize, Event)]
struct MyEvent {
    payload: String,
}
```

Generates `event_type()`, serialization, and routing support.

### `#[step]`

Turns an async function into a workflow step. Generates a `<fn_name>_registration()` function used to register the step with a `WorkflowBuilder`.

```rust
#[step]
async fn my_step(event: StartEvent, ctx: Context) -> Result<StopEvent, WorkflowError> { ... }
```

### `#[step(emits = [Type1, Type2])]`

Marks a step that can emit one of several event types. The return type becomes `Result<StepOutput, WorkflowError>`.

```rust
#[step(emits = [PositiveEvent, NegativeEvent])]
async fn route(event: AnalyzeEvent, ctx: Context) -> Result<StepOutput, WorkflowError> { ... }
```

## Events

All events live in `blazen::prelude`.

### `StartEvent`

| Field  | Type                 | Description                      |
|--------|----------------------|----------------------------------|
| `data` | `serde_json::Value`  | JSON payload passed to `run()`   |

### `StopEvent`

| Field    | Type                 | Description                      |
|----------|----------------------|----------------------------------|
| `result` | `serde_json::Value`  | Final workflow output            |

### `InputRequestEvent`

| Field        | Type                 | Description                          |
|--------------|----------------------|--------------------------------------|
| `request_id` | `String`            | Unique ID correlating request/response |
| `prompt`     | `String`            | Human-readable prompt text           |

### `InputResponseEvent`

| Field        | Type                 | Description                          |
|--------------|----------------------|--------------------------------------|
| `request_id` | `String`            | Must match the originating request   |
| `response`   | `serde_json::Value`  | Human-provided response payload      |

## WorkflowBuilder

### `WorkflowBuilder::new(name)`

```rust
fn new(name: &str) -> WorkflowBuilder
```

Create a new workflow builder with the given name.

### `.step(registration)`

```rust
fn step(self, registration: StepRegistration) -> Self
```

Add a step. Pass the `<fn_name>_registration()` function generated by `#[step]`.

### `.input_handler(handler)`

```rust
fn input_handler(self, handler: Arc<dyn Fn(InputRequestEvent) -> BoxFuture<Result<InputResponseEvent, WorkflowError>>>) -> Self
```

Register a handler invoked each time a step emits an `InputRequestEvent`.

### `.build()`

```rust
fn build(self) -> Result<Workflow, WorkflowError>
```

Finalize and return the `Workflow`. Returns an error if the configuration is invalid.

---

## Workflow

### `.run(input)`

```rust
async fn run(&self, input: serde_json::Value) -> Result<WorkflowHandler, WorkflowError>
```

Start the workflow with the given JSON input. Returns a `WorkflowHandler` for observing results and stream events.

---

## WorkflowHandler

### `.result()`

```rust
async fn result(self) -> Result<StopEvent, WorkflowError>
```

Wait for the workflow to complete and return the final `StopEvent`.

### `.stream_events()`

```rust
fn stream_events(&self) -> impl Stream<Item = Box<dyn Event>>
```

Subscribe to stream events published via `ctx.write_event_to_stream()`. The stream completes when the workflow finishes.

## Context

Passed as the second argument to every step function.

### `.set(key, value)`

```rust
fn set(&self, key: &str, value: serde_json::Value)
```

Store a value in the shared key-value store for this run.

### `.get(key)`

```rust
fn get(&self, key: &str) -> Option<serde_json::Value>
```

Retrieve a value by key. Returns `None` if the key has not been set.

### `.run_id()`

```rust
fn run_id(&self) -> String
```

Get the unique UUID for this workflow run.

### `.send_event(event)`

```rust
fn send_event(&self, event: impl Event)
```

Route an event manually instead of returning it from the step. When using `send_event`, the step returns `Result<(), WorkflowError>`.

### `.write_event_to_stream(event)`

```rust
fn write_event_to_stream(&self, event: impl Event)
```

Publish an event to the external stream without affecting the routing pipeline. Subscribers receive it via `handler.stream_events()`.

---

## StepOutput

Used as the return type for steps annotated with `#[step(emits = [...])]`.

### `StepOutput::Single(Box<dyn Event>)`

Wrap a single event for output when the step can emit multiple types.

```rust
Ok(StepOutput::Single(Box::new(MyEvent { ... })))
```
